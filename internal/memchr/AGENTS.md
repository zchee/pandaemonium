# internal/memchr KNOWLEDGE BASE

## OVERVIEW

Internal byte-search package ported from the byte-search subset of
BurntSushi `memchr`. It owns 12 APIs, architecture-specific backends,
assembly, committed golden corpus, and perf gates.

## WHERE TO LOOK

| Task | Location | Notes |
| --- | --- | --- |
| API surface | `api.go`, `doc.go`, `README.md` | Exactly 12 functions, `-1` miss sentinel. |
| Dispatch matrix | `dispatch.go`, `dispatch_swar_default.go`, `memchr_*.go` | One binding per tuple. |
| amd64 assembly | `memchr_amd64_avx2.s`, `memchr_amd64_v4.s` | AVX2 and AVX-512 artifacts. |
| arm64 assembly | `memchr*_arm64.s`, `memrchr*_arm64.s` | NEON routines. |
| SWAR fallback | `memchr_swar*.go` | Pure Go fallback and forced verification. |
| Corpus | `testdata/golden_corpus.json`, `golden_test.go` | Generated by Rust shim in `hack/`. |
| Perf gate | `bench*_test.go`, `hack/memchr-perf-gate` | Stdlib and artifact policies. |

## CONVENTIONS

- Public surface is exactly:
  - `Memchr`, `Memchr2`, `Memchr3`
  - `Memrchr`, `Memrchr2`, `Memrchr3`
  - `IndexByte`, `IndexByte2`, `IndexByte3`
  - `LastIndexByte`, `LastIndexByte2`, `LastIndexByte3`
- All functions return `-1` on miss.
- Wrapper functions stay thin; the inliner should collapse them.
- Use `simd/archsimd` for amd64 local feature decisions.
- `GOAMD64=v4` is AVX-512 primary artifact; `v3` is AVX2 fallback artifact;
  v1/v2 uses runtime AVX2-vs-SSE2 selection.
- `force_swar` is test/profile verification only.
- Golden corpus regeneration must commit `Cargo.toml`, `Cargo.lock`, and
  `testdata/golden_corpus.json` together.

## ANTI-PATTERNS

- Do not add `memmem`, iterators, stateful `Memchr*` objects, cgo, AVO, wasm32
  SIMD, or new algorithm families without a design and benchmark plan.
- Do not edit assembly or dispatch without checking the build-tag matrix.
- Do not let `GODEBUG=cpu.avx2=off` silently downgrade CI perf evidence.
- Do not promote `BenchmarkTuning...` rows into hard gates by accident.

## COMMANDS

```bash
go test -count=1 ./internal/memchr
go test -tags=force_swar -count=1 ./internal/memchr
GOEXPERIMENT=nosimd go test -count=1 ./internal/memchr
GOAMD64=v4 go test -bench=. -benchmem -run=^$ ./internal/memchr
GOAMD64=v3 go test -bench=. -benchmem -run=^$ ./internal/memchr
go run ./hack/memchr-perf-gate
go tool asmfmt -w internal/memchr/*.s
(cd hack/extract-memchr-corpus && cargo run --release) > internal/memchr/testdata/golden_corpus.json
```
